# v0.0.17 Handoff

## Recommended Implementation Order

1. Re-embedding workflow contract
   - Add an admin-triggered re-embedding request field or endpoint.
   - Select ready documents whose embeddings must be regenerated after pgvector migration.
   - Ensure failures set `embedding_status=failed` and `last_error`.

2. Migration job boundary
   - Add a Job manifest builder or admin command wrapper for `ApplyPGVectorEmbeddingMigration`.
   - Require `pgVectorMigrationApproved=true` before the job can be created.
   - Keep migration logs free of DSNs and tokens.

3. Runtime status updates
   - Update `PGVectorReady` after migration/live check evidence exists.
   - Add status for re-embedding progress.

4. Console diagnostics
   - Decide where CYOps displays `/api/ops/retrieval-metrics`.
   - Keep CYOps diagnostics separate from the Red Hat Lightspeed viewer.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- Silent database migration.

## Acceptance Criteria

- Re-embedding is explicitly admin-triggered.
- Migration job creation requires approval.
- Runtime status reflects pgvector and re-embedding progress.
- Console diagnostics entry point is specified.
