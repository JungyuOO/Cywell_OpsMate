# v0.0.6 Handoff

## 우선 구현 순서

1. PostgreSQL repository implementation
   - documents list/create/get/delete marker
   - chat sessions/messages
   - migration application strategy

2. Real document storage adapter
   - PVC-backed write path
   - file size and extension validation
   - sanitized object URI persistence

3. Real Lightspeed REST API client
   - endpoint and Secret loading from `OpsMateConfig`
   - request/response mapping
   - provider timeout and sanitized errors

4. Initial frontend bundle
   - CYOps launcher
   - chat drawer shell
   - document upload/list panel
   - API client integration

5. Ingestion/embedding planning
   - parser/chunker interface
   - embedding worker Deployment boundary
   - pgvector migration update

## v0.0.6에서 하지 않을 것

- production HA PostgreSQL
- full parser coverage for every enterprise file format
- full vector search tuning
- Operator bundle/catalog release
- Red Hat Lightspeed ConsolePlugin hiding/deletion

## Acceptance criteria

- Backend can persist document metadata to PostgreSQL.
- Upload can write to the configured storage boundary.
- Chat can route to a real Lightspeed client behind an interface.
- Frontend skeleton can call backend API endpoints.
- v0.0.7 scope is clear for ingestion and embeddings.
