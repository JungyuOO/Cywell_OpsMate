# v0.0.5 Handoff

## 우선 구현 순서

1. Backend API skeleton
   - `/api/chat`
   - `/api/documents`
   - `/api/documents/{documentId}`
   - request/response DTO와 handler tests

2. PostgreSQL schema/migration skeleton
   - documents
   - chunks
   - embeddings placeholder
   - chat sessions/messages

3. Document upload storage boundary
   - local PVC path abstraction first
   - object store adapter interface only

4. Lightspeed provider client skeleton
   - endpoint/Secret config loading
   - request/response interface
   - no customer document leakage by default

5. ConsolePlugin frontend bundle skeleton
   - CYOps launcher/drawer shell
   - chat composer
   - document list/upload panel placeholder

## v0.0.5에서 하지 않을 것

- full embedding model runtime
- full parser support for every document format
- production PostgreSQL HA
- Operator bundle/catalog release
- Red Hat Lightspeed ConsolePlugin hiding/deletion

## Acceptance criteria

- API handlers compile and have unit tests.
- Document upload can be accepted and metadata can be represented.
- Chat endpoint can route to a mocked provider.
- UI bundle skeleton can be served or packaged in a predictable path.
- v0.0.6 scope is clear for real ingestion/embedding.
