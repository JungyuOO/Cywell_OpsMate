# v0.0.18 Handoff

## Recommended Implementation Order

1. Migration execution entrypoint
   - Add a real command/CLI entrypoint for `cyops-pgvector-migrate`.
   - It should call `ApplyPGVectorEmbeddingMigration` and avoid logging DSNs or tokens.
   - Keep Job creation gated by `pgVectorMigrationApproved=true`.

2. Admin-triggered re-embedding
   - Add an admin API endpoint or Kubernetes Job builder that invokes `ReembedReadyDocuments`.
   - Include a limit/batch size.
   - Surface processed and failed counts without document text.

3. Runtime status updates
   - Update `PGVectorReady` after migration/live check evidence exists.
   - Add re-embedding progress status.

4. Console diagnostics
   - Decide where CYOps displays `/api/ops/retrieval-metrics`.
   - Keep CYOps diagnostics separate from the Red Hat Lightspeed viewer.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- Silent database migration.

## Acceptance Criteria

- Migration Job has a real executable entrypoint.
- Re-embedding can be triggered by an admin workflow.
- Runtime status reflects migration and re-embedding progress.
- Console diagnostics entry point is specified.
