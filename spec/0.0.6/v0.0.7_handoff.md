# v0.0.7 Handoff

## 우선 구현 순서

1. Appserver dependency wiring
   - environment/config based PostgreSQL DSN
   - document storage base path
   - Lightspeed endpoint/provider configuration

2. Upload handler persistence
   - store uploaded file through `DocumentStorage`
   - write metadata through `PostgresDocumentRepository`
   - persist object URI

3. Runtime migration strategy
   - explicit migration command or appserver startup migration gate
   - idempotent migration execution
   - failure reporting

4. Ingestion planning to implementation
   - parser interface
   - chunker implementation
   - ingestion job/worker state model

5. Embedding boundary
   - embedding provider interface
   - pgvector `VECTOR(n)` migration update after dimensions are fixed
   - query-time retrieval API

## v0.0.7에서 하지 않을 것

- full production-grade vector tuning
- every document format parser
- HA PostgreSQL operations
- Operator bundle/catalog release
- hiding or deleting Red Hat Lightspeed ConsolePlugin

## Acceptance criteria

- Appserver can be configured with PostgreSQL DSN and storage path.
- Upload can write a file and persist metadata.
- Integration tests cover repository plus storage flow.
- Migration execution strategy is explicit.
- v0.0.8 scope is clear for ingestion and embeddings.
