# OpenShift pgvector Migration Runbook

## Preconditions

- `OpsMateConfig.spec.embedding.dimensions` is fixed for the selected embedding model.
- PostgreSQL backup is available and restore procedure is tested.
- The database image provides the `vector` extension, for example `pgvector/pgvector:pg16` or an approved internal mirror.
- The appserver has a Secret containing the DSN:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: opsmate-postgres-dsn
type: Opaque
stringData:
  dsn: postgres://opsmate:<password>@sample-postgres.opsmate.svc:5432/opsmate?sslmode=disable
```

## OpsMateConfig Settings

```yaml
spec:
  database:
    type: postgres
    image: pgvector/pgvector:pg16
    dsnSecretRef: opsmate-postgres-dsn
    dsnSecretKey: dsn
    pgVectorMigrationApproved: false
  embedding:
    dimensions: 768
    requirePGVector: true
    retrievalMode: pgvector
```

## Validation Steps

1. Apply the CRD and `OpsMateConfig`.
2. Confirm the Postgres Deployment uses a pgvector-enabled image.
3. Confirm the appserver Deployment has `CYOPS_POSTGRES_DSN` from SecretKeyRef.
4. Confirm appserver startup fails with `pgvector extension is not ready` when the extension is unavailable.
5. After backup and approval, set `pgVectorMigrationApproved: true` and apply the controlled migration generated by `PGVectorEmbeddingMigrationSQL(dimensions)` or the admin tool that wraps `ApplyPGVectorEmbeddingMigration`.
6. Re-embed ready documents after migration because BYTEA fallback embeddings are reset by the conversion path.
7. Run:

```powershell
$env:CYOPS_PGVECTOR_TEST_DSN = "postgres://opsmate:<password>@localhost:<port>/opsmate?sslmode=disable"
go test ./internal/appserver -run TestPGVectorLiveRAGSmoke -count=1 -v
```

8. Query `/api/ops/retrieval-metrics` and confirm `byMode.pgvector` increments.

## Rollback Notes

- For dev/test, switch `retrievalMode` back to `bytea` and restore a pre-migration backup.
- For production, restore the backup first; do not attempt lossy reverse conversion from `VECTOR(n)` to BYTEA without an explicit tested script.
- Keep Red Hat Lightspeed plugin resources untouched; CYOps remains a separate ConsolePlugin.
