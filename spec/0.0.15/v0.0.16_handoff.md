# v0.0.16 Handoff

## Recommended Implementation Order

1. Migration approval surface
   - Add an explicit admin-controlled migration trigger or status field.
   - Reconcile must not apply pgvector migration silently.

2. Status conditions
   - Add conditions for `PostgresDSNConfigured`, `PGVectorRequired`, `PGVectorReady`, and `RetrievalModeReady`.
   - Surface startup/migration failures without including DSNs or tokens.

3. Re-embedding workflow
   - Add a controlled re-embedding job/admin trigger after pgvector migration.
   - Failed documents should set `embedding_status=failed` and `last_error`.

4. Console diagnostics
   - Decide where CYOps should show retrieval metrics.
   - Keep diagnostics separate from the Red Hat Lightspeed viewer.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- Automatic destructive database migration.

## Acceptance Criteria

- Migration requires explicit admin approval.
- Operator status exposes pgvector readiness and DSN configuration state.
- Re-embedding after migration is documented or executable.
- Console diagnostic entry point is specified.
