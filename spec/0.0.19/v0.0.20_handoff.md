# v0.0.20 Handoff

## Recommended Implementation Order

1. OpenShift migration Job execution
   - Apply `PGVectorMigrationJob` in a target namespace.
   - Verify `cyops-pgvector-migrate` runs to completion.
   - Confirm Job logs do not expose DSNs or tokens.

2. Runtime status updates
   - Update `status.pgVectorReady` after successful migration/live smoke.
   - Update `status.reembedding` after admin re-embedding calls or jobs.

3. Production admin authorization
   - Replace or wrap token header with OpenShift OAuth/RBAC.
   - Restrict `/api/ops/reembed` to operator admins.

4. Console diagnostics
   - Add CYOps admin/debug panel placement for retrieval metrics and re-embedding status.
   - Keep the Red Hat Lightspeed viewer untouched and separate.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- Silent database migration.

## Acceptance Criteria

- Migration Job runs in OpenShift with the real CLI.
- Runtime status reflects migration and re-embedding evidence.
- Admin authorization design is production-ready.
- Console diagnostics entry point is specified.
