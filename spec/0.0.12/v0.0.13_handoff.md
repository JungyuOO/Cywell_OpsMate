# v0.0.13 Handoff

## 우선 구현 순서

1. OpenShift live pgvector validation
   - pgvector-enabled PostgreSQL image를 OpenShift namespace에 배포한다.
   - `CYOPS_PGVECTOR_REQUIRED=true`와 `CYOPS_RETRIEVAL_MODE=pgvector`로 appserver startup을 검증한다.
   - `PGVectorEmbeddingMigrationSQL(dimensions)` output을 controlled migration으로 적용한다.

2. Re-embedding rollout
   - migration 후 기존 `BYTEA` fallback embeddings를 reset한다.
   - ready documents를 대상으로 re-embedding job 또는 admin-triggered flow를 정의한다.
   - 실패 document는 `embedding_status=failed`와 `last_error`로 관찰한다.

3. SQL similarity verification
   - `/api/chat` RAG path가 pgvector SQL-ranked chunks를 반환하는 live smoke test를 추가한다.
   - fallback `bytea` mode는 dev/test mode로만 남긴다.

4. Metrics backend
   - `RetrievalObserver`를 concrete metrics/log sink에 연결한다.
   - provider latency, retrieval duration, slow retrieval count, failure reason count를 노출한다.

## v0.0.13에서 하지 않을 것

- production ANN tuning 전체
- 모든 embedding provider 구현
- BYOKnowledge 기반 indexing
- Red Hat Lightspeed ConsolePlugin 숨김/삭제
- 별도 embedding server HA 운영

## Acceptance criteria

- OpenShift pgvector-enabled DB에서 startup readiness가 통과한다.
- controlled `VECTOR(n)` migration과 re-embedding 절차가 검증된다.
- `/api/chat` RAG path가 pgvector SQL-ranked citations를 반환한다.
- retrieval/embedding 관측값이 metrics 또는 structured log sink로 전달된다.
