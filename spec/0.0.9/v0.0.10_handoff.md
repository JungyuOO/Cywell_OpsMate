# v0.0.10 Handoff

## 우선 구현 순서

1. pgvector readiness and migration
   - target embedding dimension을 확정한다.
   - `CREATE EXTENSION IF NOT EXISTS vector` 실행 가능 여부를 startup validation에 반영한다.
   - `cyops_document_embeddings.embedding`을 `BYTEA`에서 `VECTOR(n)`으로 전환하는 migration을 추가한다.

2. External embedding provider configuration
   - provider endpoint, model, dimension, Secret reference를 env/config boundary에 추가한다.
   - disconnected OpenShift 환경에서는 internal service endpoint도 지원한다.
   - provider failure와 retry policy는 document `embedding_status`와 `last_error`로 관찰 가능해야 한다.

3. Retrieval ranking
   - `PostgresRetriever`를 vector similarity query로 확장한다.
   - `documentIds` filter와 top-k limit을 유지한다.
   - retrieved context text는 bounded length로 provider request에 전달한다.

4. Chat response citations
   - citations에 score 또는 rank field 추가 여부를 결정한다.
   - UI가 source document/chunk를 표시할 수 있도록 backend contract를 보강한다.

## v0.0.10에서 하지 않을 것

- production ANN index tuning 전체
- 모든 embedding provider 구현
- BYOKnowledge 기반 indexing
- Red Hat Lightspeed ConsolePlugin 숨김/삭제
- 별도 embedding server HA 운영

## Acceptance criteria

- pgvector extension readiness가 명확한 error로 드러난다.
- configured embedding provider가 chunk embeddings를 생성한다.
- `/api/chat` RAG path가 vector-ranked chunks를 provider context로 전달한다.
- citations가 retrieval source를 안정적으로 표현한다.
