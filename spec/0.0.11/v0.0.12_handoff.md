# v0.0.12 Handoff

## 우선 구현 순서

1. pgvector runtime activation
   - pgvector-enabled PostgreSQL image 또는 OpenShift DB에서 `CheckPGVectorReady`를 검증한다.
   - `PGVectorEmbeddingMigrationSQL(dimensions)` output을 controlled migration path로 적용한다.
   - migration 후 기존 chunks를 re-embed하는 운영 절차를 확정한다.

2. SQL similarity retriever
   - `PostgresRetriever`에 pgvector SQL ranking mode를 추가한다.
   - `documentIds`, namespace isolation, top-k limit을 유지한다.
   - BYTEA fallback ranking은 dev/test mode로 제한한다.

3. Embedding auth operations
   - `CYOPS_EMBEDDING_TOKEN`은 Authorization header로만 사용하고 logs/status에는 남기지 않는다.
   - Secret rotation 후 appserver restart/reload 전략을 정리한다.

4. Observability
   - embedding provider latency/failure와 retrieval query time을 metric/log boundary로 노출한다.
   - slow retrieval threshold와 failure reason taxonomy를 정리한다.

## v0.0.12에서 하지 않을 것

- production ANN tuning 전체
- 모든 embedding provider 구현
- BYOKnowledge 기반 indexing
- Red Hat Lightspeed ConsolePlugin 숨김/삭제
- 별도 embedding server HA 운영

## Acceptance criteria

- pgvector-enabled DB에서 `VECTOR(n)` migration이 검증된다.
- `/api/chat` RAG path가 pgvector SQL-ranked chunks를 provider context로 전달한다.
- fallback ranking과 pgvector ranking mode가 config로 분리된다.
- embedding auth와 token rotation 운영 조건이 문서화된다.
