# v0.0.11 Handoff

## 우선 구현 순서

1. pgvector schema migration
   - target embedding dimension을 runtime config와 migration 문서에서 하나로 고정한다.
   - `cyops_document_embeddings.embedding`을 `VECTOR(n)`으로 전환하는 migration을 추가한다.
   - 기존 `BYTEA` fallback data migration 또는 reset 전략을 명확히 한다.

2. pgvector SQL retrieval
   - `PostgresRetriever`를 SQL similarity ranking으로 확장한다.
   - `documentIds` filter, namespace isolation, top-k limit을 유지한다.
   - fallback BYTEA ranking은 pgvector readiness가 false인 dev/test path로만 둔다.

3. Embedding provider auth
   - embedding endpoint credential Secret reference를 `OpsMateConfig` 또는 Deployment env boundary에 추가한다.
   - appserver는 token value를 log/response에 노출하지 않는다.
   - disconnected cluster internal service endpoint를 같은 provider interface로 지원한다.

4. Index and operations
   - exact search 먼저 검증한 뒤 ANN index tuning을 별도 phase로 분리한다.
   - index creation failure와 slow query observability를 문서화한다.

## v0.0.11에서 하지 않을 것

- 모든 embedding provider 구현
- production ANN tuning 전체
- BYOKnowledge 기반 indexing
- Red Hat Lightspeed ConsolePlugin 숨김/삭제
- 별도 embedding server HA 운영

## Acceptance criteria

- pgvector `VECTOR(n)` schema path가 검증된다.
- `/api/chat` RAG path가 pgvector-ranked chunks를 provider context로 전달한다.
- embedding provider auth Secret boundary가 문서와 코드에 반영된다.
- fallback ranking과 pgvector ranking의 운영 조건이 명확하다.
