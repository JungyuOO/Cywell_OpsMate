# v0.0.8 Handoff

## 우선 구현 순서

1. Ingestion job state model
   - `cyops_documents.status`를 `uploaded -> processing -> ready|failed`로 전환한다.
   - 실패 시 `last_error`를 기록한다.
   - appserver 내부 synchronous worker로 시작하고, 별도 Deployment 분리는 후속 조건이 생길 때 검토한다.

2. Parser interface
   - `Parser` interface는 `Parse(ctx, objectURI) ([]ParsedSection, error)` 형태로 시작한다.
   - v0.0.8에서는 plain text와 markdown만 구현한다.
   - PDF, docx, xlsx는 별도 parser dependency 검토 후 후속 버전에서 추가한다.

3. Chunker implementation
   - section text를 deterministic chunk로 분할한다.
   - chunk index, source_start, source_end를 `cyops_document_chunks`에 저장한다.
   - token count는 초기에는 byte/rune 기반 estimate로 기록하고, embedding model 확정 후 tokenizer로 보강한다.

4. Embedding boundary
   - `EmbeddingProvider` interface를 추가한다.
   - v0.0.8에서는 provider mock과 storage schema 경계까지만 구현한다.
   - pgvector `VECTOR(n)` 전환은 embedding dimension을 확정한 뒤 진행한다.

5. Retrieval API
   - `/api/chat` RAG option이 document IDs를 받을 때 retrieval interface를 호출하는 경계를 만든다.
   - 실제 vector similarity ranking은 v0.0.9 이후로 넘길 수 있다.

## v0.0.8에서 하지 않을 것

- 모든 문서 형식 parser 구현
- production vector tuning
- 별도 embedding server Deployment
- Red Hat Lightspeed ConsolePlugin 숨김/삭제
- BYOKnowledge 연동

## Acceptance criteria

- 업로드된 문서를 ingestion 대상으로 전환할 수 있다.
- plain text/markdown 문서를 chunk table에 저장할 수 있다.
- ingestion 실패가 document status와 `last_error`에 반영된다.
- embedding provider interface와 mock test가 있다.
- v0.0.9 retrieval/vector scope가 명확하다.
