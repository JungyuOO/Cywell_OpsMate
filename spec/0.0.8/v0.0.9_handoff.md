# v0.0.9 Handoff

## 우선 구현 순서

1. Embedding provider interface
   - `Embed(ctx, chunks []DocumentChunk) ([]EmbeddingVector, error)` 형태로 시작한다.
   - 초기 구현은 deterministic mock provider로 테스트를 고정한다.
   - 실제 model/provider 선택은 OpenShift disconnected 환경과 고객 BYO endpoint 요구를 고려해 별도 config로 둔다.

2. pgvector migration
   - embedding dimension을 확정한 뒤 `cyops_document_embeddings.embedding`을 `BYTEA`에서 `VECTOR(n)`으로 전환한다.
   - extension 설치 권한이 없는 환경을 위해 startup validation error를 명확히 한다.
   - v0.0.9에서 바로 production index tuning까지 하지 않는다.

3. Embedding persistence
   - ingestion 완료 후 chunk별 embedding을 생성하고 `cyops_document_embeddings`에 저장한다.
   - embedding 실패는 document `embedding_status=failed`와 `last_error`로 표시한다.
   - chunk text는 provider로 전달하되 chat provider context에는 원문 전체를 직접 넣지 않는다.

4. Retrieval API
   - `Retriever` interface를 추가한다.
   - `/api/chat`의 `rag.enabled=true`와 `documentIds`가 있을 때 retrieval context를 provider request에 붙인다.
   - citation은 document id, filename, chunk id를 포함한다.

## v0.0.9에서 하지 않을 것

- production ANN index tuning
- 모든 embedding provider 구현
- 별도 embedding server Deployment
- BYOKnowledge 기반 indexing
- Red Hat Lightspeed ConsolePlugin 숨김/삭제

## Acceptance criteria

- deterministic embedding mock으로 chunk embedding persistence를 검증한다.
- pgvector migration 전환 조건이 코드와 문서에 반영된다.
- `/api/chat` RAG path가 retrieval interface를 호출한다.
- citations가 chat response에 포함된다.
