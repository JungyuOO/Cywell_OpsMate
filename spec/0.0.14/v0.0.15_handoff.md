# v0.0.15 Handoff

## Recommended Implementation Order

1. OpenShift pgvector database deployment validation
   - Deploy or bind a pgvector-enabled PostgreSQL instance in the target namespace.
   - Store DSN through Secret-backed appserver configuration.
   - Run `CYOPS_PGVECTOR_TEST_DSN` live smoke against the OpenShift route or port-forwarded service.

2. Operator migration runbook
   - Define how an operator admin approves `ApplyPGVectorEmbeddingMigration`.
   - Record backup, rollback, and re-embedding expectations.
   - Keep default reconciliation from applying pgvector migration implicitly.

3. Appserver startup validation in cluster
   - Verify `CYOPS_PGVECTOR_REQUIRED=true`.
   - Verify `CYOPS_RETRIEVAL_MODE=pgvector`.
   - Confirm startup failure message when the extension is missing.

4. ConsolePlugin diagnostic surface
   - Decide whether `/api/ops/retrieval-metrics` should be displayed in the CYOps admin/debug panel.
   - Keep customer prompts and document contents out of diagnostics.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- High availability for a separate embedding server.

## Acceptance Criteria

- OpenShift pgvector service is validated with the v0.0.14 live smoke.
- Migration runbook is documented with explicit approval and rollback notes.
- Cluster appserver startup passes with pgvector-required settings.
- Retrieval metrics are visible during the cluster smoke test.
