# v0.0.19 Handoff

## Recommended Implementation Order

1. Runtime status updates
   - Update `PGVectorReady` based on migration/live check evidence.
   - Add re-embedding progress fields or conditions.
   - Keep DSNs, tokens, prompts, filenames, and document text out of status.

2. Admin endpoint authorization
   - Define OpenShift/RBAC policy for `/api/ops/reembed`.
   - Ensure only operator admins can trigger re-embedding.

3. Migration Job execution validation
   - Run `PGVectorMigrationJob` in OpenShift with the real `cyops-pgvector-migrate` command.
   - Verify logs do not expose DSN or token values.

4. Console diagnostics
   - Decide where CYOps displays `/api/ops/retrieval-metrics` and re-embedding status.
   - Keep this separate from the Red Hat Lightspeed viewer.

## Out Of Scope

- ANN index tuning.
- BYOKnowledge indexing.
- Red Hat Lightspeed plugin hiding or deletion.
- Silent database migration.

## Acceptance Criteria

- Operator/runtime status reflects migration and re-embedding evidence.
- Admin endpoints have an authorization design.
- Migration Job is validated in OpenShift.
- Console diagnostic entry point is specified.
